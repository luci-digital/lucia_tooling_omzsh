## Build System (Nix + just)

### Primary Commands

All commands use Nix for reproducibility. Run via `nix run .#<app>` or use the `just` convenience wrapper:

```bash
# Development
just dev                 # Start frontend dev server (HMR on port 3000)
just dev-shell           # Enter full Nix dev shell

# Testing & Validation
just check               # Full verification gate (tests + lint + zsh + compose)
just ci                  # Broader CI gate (includes web build)
just flake-check         # nix flake check (builds + tests luci-vcs)
just fmt                 # Format Nix + shell + Rust in place

# Individual checks
just cargo-test          # Rust VCS substrate tests
just cargo-check         # Rust check only
just web-test            # Frontend Vitest tests
just web-build           # Production frontend build
just shellcheck          # Lint shell scripts
just shfmt               # Check shell formatting
just zsh-test            # Validate zsh config + syntax
just compose-config      # Validate Podman compose file

# Stack Operations
just init-env local      # Scaffold .env from template (local|staging|production)
just up                  # Start Podman stack (detached)
just down                # Stop Podman stack
just deploy-local        # Build images + start + show status
```

### Why Nix?

Every command routes through `flake.nix` for reproducibility. Behavior is identical in CI, dev shells, and local execution. The `justfile` is a thin convenience layer.

## Known Limitations

1. **Nix/Podman not fully validated** — Authored in an environment without nix/podman/zsh. `flake.nix`, compose edits, and zsh config are untested. Rust tests (16 pass) and web build are verified.
2. **flake.lock missing** — Run `nix flake lock` on a Nix host first.
3. **Impure web build** — Uses Nix-provided node+pnpm via `nix run .#web-build`. Sandboxed build needs `pnpm.fetchDeps` hash.
4. **curl|sh installers** — Three Dockerfiles use piped installers (nodesource, code-server, xmake). Flagged in security audit.
5. **Optional tools** — `foundationdb`, `kubo` load only if nixpkgs attr resolves on host.

## Common Tasks

### Add a new frontend route
1. Create `modules/web/luci-frontend/src/routes/<name>.tsx`
2. Router auto-generates `src/routeTree.gen.ts` on next dev/build
3. Add server functions in `src/functions/` if needed

### Add a Podman service
1. Add service definition to `modules/orchestration/podman/podman-compose.yml`
2. Update `modules/docs/orchestration/services.md`
3. Test: `just compose-config && just up`

### Modify zsh config
1. Edit `modules/shell/zshrc/.zshrc`
2. Test: `just zsh-test`
3. Enter shell: `nix develop .#shell` then `zsh`

### Update Rust VCS
1. Edit `modules/scm/luci-vcs/src/`
2. Test: `just cargo-test`
3. Format: `cd modules/scm/luci-vcs && cargo fmt`
